基础知识

XAML

  1. 它并不是WPF唯一的布局方式，我们可以通过代码构造WPF窗口，且：Microsoft Expression Design 可精修图形界面、ExpressionBlen等工具可构建精美动画。
  2. 标记扩展： 动态的设置属性或引用其他属性（使用资源和数据绑定时出现较多）
  3. XAML布局中的元素，都将在cs代码中以【新建textBox = new TextBox】、【添加 grid.Add(textBox)】的方式放到布局中。
  4. <…> ContentProperty < /…>
  5. XAML的空白（空格）保留：需要的控件添加 xml:space=“preserve”
  6. textBox.Text = “Hello !” 而非 textBox.setText(“Hello !”);
  7. 自动跳过了【2.5 加载和编译XAML】

布局

  1. 如果想接收鼠标事件，Background值应非空。
  2. button.Margin = new Thickness(5);
  3. Border控件，继承自Controls.Decorator，常跟布局一起使用，貌似是包裹着布局的，是一种装饰。比如，Button使用ButtonChrome装饰元素，ListBox使用ListBoxChrome

布局容器

  • StackPanel : LinearLayout
  • WrapPanel ：多行的StackPanel，一行显示不下就放到第二行、第三行。。。
  • DockPanel ：贴边。
  • Grid ：乍一用是根据边距定位，其实可以弄成GridLayout的形式（并不是指定row和column的数量，而是通过代码一步步添加行列），且尺寸可按weight更改。
  • UniformGrid 这个跟GridLayout一样，通过Row=2这样设置Grid的行列数，简单粗暴。But，大部分时候，大家不用这个，而使用更加Magic的Grid布局，因为Grid可以很花哨？
  • Canvas : 使用绝对坐标，弃用该方法

其他

GridSplitter用在Grid中，可拖拽，改变Grid的布局，很好玩

通过SharedSizeGroup来进行尺寸联动，注意或许需要IsSharedSizeScope为True 。 具体可以联动哪些？尺寸吧

控件

  • System.Windows.Control
  • 接收焦点、鼠标、键盘等输入元素

内容控件

继承自ContentControl

ButtonBase, Label, CheckBox, Window等(还有HeaderedContentControl，带标题的内容)

1. Label的Target属性，可以使用快捷键去定位

<Label Taget="{Binding ElementName=txtA}">Choose _A</Lable>
<TextBox Name="txtA"></TextBox>

例如上述代码，当使用Alt+A时，光标自动定位到txtA的输入控件。

2. Button的IsCancel和IsDefault功能很有意思

3. RepeatButton：按下不松手，连续触发；Button：按下不松手，触发一次

4. RadioButton用法与安卓类似，需要注意的是，两组间若组名GroupName相同，则可以相互影响？？？

5. ToolTip既是控件，也是元素的属性，起到提示作用，自定义度很高，很给力

6. Popup，老朋友了，相比较ToolTip貌似要死板一点，必须手动显示。另外，Popup可用于交互，当鼠标离开时，Popup并不消失

特殊容器

  • ScrollViewer 允许包含的内容参与滚动过程（花式滚动，多半应用到特殊场景）
  • GroupBox 就是带标题的GroupView
  • TabItem 选项卡，TabControl里的一页
  • Expander 顾名思义，可展开折叠。
后三者都继承自HeaderedContentControl类，属于带有标题的容器

文本控件

TextBox, RichTextBox, PasswordBox三种。

TextBox
AcceptsTabs：true--确保输入时可使用回车和Tab键
LineUp(), PageUp(), ScrollToHome()等方法帮助滚动
PasswordBox

没啥说的。。。

基于范围的控件

ScrollBar
 ScrollViewer控件右侧的拖动条，一般用不到了，都直接用ScrollViewer
ProgressBar

Slider
