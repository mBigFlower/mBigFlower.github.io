ListBox显示列表。其他布局显示ListBox中选中的条目。注意StackPanel中的DataContext

<StackPanel DataContext="{Binding ElementName=listBox, Path=SelectedItem}">
    <TextBox MinHeight="60" Background="AliceBlue" Text="{Binding Path=ImgPath}"></TextBox>
    <TextBox MinHeight="40" Background="Aqua" Text="{Binding Path=Name}"></TextBox>
    <Button Name="showBtn" Content="Show" Click="showBtn_Click"/>

    <ListBox Name="listBox" 
            SelectionChanged="lstProducts_SelectionChanged"></ListBox>
</StackPanel>

该DataContext中的ElementName=（ListBox的Name，而不是 我们在cs文件中的数据） 这类似于元素的相互绑定，而不是把ListBox中的数据和StackPanel的元素绑定。

截胡

在Window的主函数里，拦截一些系统定义的事件。比如下面两个例子。

全屏

public AutoPlayWindow()
{
    InitializeComponent();

    this.Loaded += Window_Loaded; // 截胡
}
// 定义全屏事件。
private void Window_Loaded(object sender, RoutedEventArgs e)
{
    // 设置全屏    
    this.WindowState = System.Windows.WindowState.Normal;
    this.WindowStyle = System.Windows.WindowStyle.None;
    this.ResizeMode = System.Windows.ResizeMode.NoResize;
    this.Topmost = true;

    this.Left = 0.0;
    this.Top = 0.0;
    this.Width = System.Windows.SystemParameters.PrimaryScreenWidth;
    this.Height = System.Windows.SystemParameters.PrimaryScreenHeight;
}

监听键盘按键

public AutoPlayWindow()
{
    InitializeComponent();

    this.KeyDown += ModifyPrice_KeyDown; // 截胡
}

// 监听键盘的按下，根据e.Key的值，来。。。
private void ModifyPrice_KeyDown(object sender, KeyEventArgs e)
{
    if (e.Key == Key.Escape)//Esc键  
    {
        this.Close();
    }
}

    // 处理回调后的逻辑

关于命令的BUG

想给Button添加一个命令，来代替Click事件，XAML中如下操作：

 <Button Command="{Binding NormalCommand}"/>

实际跑起来没效果。F5调试后错误代码如下：

System.Windows.Data Error: 40 :
   BindingExpression path error: 'ParamCommand' property not found on 'object' ''PictureModel' (HashCode=59656956)'. 
   BindingExpression:Path=ParamCommand; 

DataItem='PictureModel' (HashCode=59656956);
target element is 'Button' (Name='ToLeftBtn'); 
target property is 'Command' (type 'ICommand')

它说在’object’ '‘PictureModel’ (HashCode=59656956)'上无法找到ParamCommand
问题是我命令的绑定跟它Model有啥关系啊？后来发现，该Button的父控件有一个数据绑定：

<Grid DataContext="{Binding ElementName=picListLB, Path=SelectedItem}">
 ...

把这个DataContext去掉后，命令就OK了。

那怎么处理两者的兼容性呢？
